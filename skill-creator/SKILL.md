# 如何创建技能

## 什么是技能
把做过的事整理成可复用的方法，下次直接调用，不用重新研究

## 标准结构
```
技能名/
└── SKILL.md    # 必需，包含name+description+用法
```

## 命名规范
- 小写字母、数字、连字符
- 简短，动词开头
- 例：`cron-ai-news-podcast`, `github-token-setup`

## SKILL.md格式
```yaml
---
name: 技能名
description: 描述(做什么+什么时候用)
---
# 详细说明

## 用法
命令/步骤

## 常见问题
| 问题 | 解决 |
|------|------|
| xxx | yyy |
```

## 创建步骤
1. 遇到问题 → 自己解决
2. 解决完写成技能（中文，简洁）
3. 存到 `~/.openclaw/workspace/skills/custom/`
4. 同步到GitHub

## 技能市场
- clawhub.com — 5400+ 技能
- github.com/openclaw/skills — 6300+ 官方技能

找不到现成的再自己写！

## ⚠️ 重要原则：辨别与筛选

### 安全性
- 查作者可信度（官方/大厂 vs 陌生人）
- 看代码是否要执行敏感操作（写文件、执行命令、上传数据）
- 避免来路不明的脚本

### 效率
- 看更新时间（长期不更新的可能已过时）
- 看Star数/下载量（群众眼光雪亮）
- 简单测试再正式用

### 质量
- 代码是否规范（不是屎山）
- 文档是否清晰
- 是否有明显bug
