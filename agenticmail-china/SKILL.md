# 中国内地网络限制：邮件服务

## 整体情况

### 1. 端口封锁
| 端口 | 协议 | 状态 |
|------|------|------|
| 25 | SMTP | 被封锁（阿里云/腾讯云默认禁用） |
| 465 | SMTPS | 可能可用，但不稳定 |
| 587 | SMTP | 可能可用 |
| 993 | IMAP | 被封锁（Gmail等） |
| 995 | POP3 | 可能可用 |

### 2. 国外邮件服务
| 服务 | 中国可用？ | 说明 |
|------|-----------|------|
| Gmail | ❌ | IMAP被封，必须翻墙 |
| Outlook | ❌ | 同上 |
| Yahoo | ❌ | 同上 |
| iCloud | ❌ | 同上 |

### 3. 国内邮件服务
| 服务 | 说明 |
|------|------|
| QQ邮箱 | 正常 |
| 163邮箱 | 正常 |
| 阿里云邮箱 | 正常 |
| 企业邮箱 | 正常 |

---

## 对AgenticMail的影响

### 在中国内地部署
- ❌ 无法使用Gmail/Outlook等国外邮箱
- ❌ 25端口被封锁，发邮件困难
- ⚠️ 可以用国内邮箱，但AgenticMail主要面向国外服务

### 在海外部署（推荐）
- ✅ Gmail/Outlook正常使用
- ✅ 25/465/587端口正常
- ✅ 不受限制

---

## 解决方案

### 方案1：只在海外部署
- 我（海外）部署AgenticMail
- 本地需要国外网站时 → 发消息给我（飞书/邮件）→ 我去抓
- **不需要Gmail账号也能用**

### 方案2：域名模式
- 买一个域名（~10/年）
- 不需要Gmail，用自己的域名邮箱
- 两个机器人都用 @自己域名.com

### 方案3：API中转
- 我提供REST API
- 本地通过HTTP请求我
